Oldest left, newest right
Arrival order, never re-sorted. The leftmost dot is what's playing now (Terminal A's first clip of 3).
Run clustering
Consecutive clips from the same terminal sit side-by-side. A 12 px gap is inserted only when the next clip's session differs — so the timeline reads as A A A — B B B — C C without any colour confusion.
Colour = session identity
Each terminal picks one of the 24 arrangements (8 solids + 8 horizontal splits + 8 vertical splits). Playing dots pulse with a white ring; the base colour never changes so you always know who is talking.
When this hurts you
If Terminal C has the important message but Terminal A is rambling 3 clips deep, you'd wait through all of A + B first. That's what the focus-star on the sessions panel (shot 04) is for.

How to read the strip
Oldest on the left, newest on the right — always arrival order, never re-sorted. Runs of same-colour dots = clips from one terminal; a 12 px gap marks a change of speaker. So this strip says: A sent 3, B sent 2, C sent 2, plus one highlight-to-speak.
1 · Heard (first two reds)
Already played. The dot stays in place so you can click it to replay or right-click to delete. It disappears on its own after the auto-prune window (3–600 s, default 20 s). Flip auto-prune off in Playback if you want clips to stack up for review instead.
2 · Playing (third red)
Slightly larger, pulsing white ring. The base colour stays the session colour — the ring is layered on top, not a replacement — so you always know which terminal is currently talking.
3 · Queued (yellows + greens)
Plain flat discs. When the current red ends, Terminal B's two yellows play next (oldest first), then Terminal C's two greens. Unless a focus ★ is set (shot 04) or a hey-jarvis clip arrives, which jumps to the front.
4 · J-clip (blue, on the right)
Highlight-to-speak clip — you selected text and said "hey jarvis" or pressed Ctrl+Shift+S. The small J glyph marks it. Colour inherits from the Claude Code terminal the selection came from (blue here); grey if triggered from outside a Claude session (browser, PDF, Slack). J-clips have the highest playback priority — they jump the entire queue.
Muted sessions never appear
When a session is 🔇-muted in the panel, its clips are dropped at arrival — no synthesis, no file on disk, no dot, no audio. The statusline in that terminal shows a 🔇 prefix so you remember. Unmuting resumes on the next clip the terminal produces.
Click / right-click
Left-click any dot to (re)play it manually. Right-click to remove it from the queue immediately without waiting for auto-prune. Both work on heard clips and queued clips.
Clear all
The ✕ (trash) button in the controls row wipes every clip at once — useful when three chatty terminals fire at the same time and you just want quiet. Held clips (currently playing) finish first; the rest drop.

Playback — speed + auto-prune
Speed slider 0.5–2.5×. Auto-prune toggle + seconds (3–600, default 20) — when on, played clips disappear from the strip after that many seconds. Flip off if you're stepping away and want clips to stack up for review.
Sessions row · 7 columns
Chevron · swatch · 8-char session ID · editable label · colour dropdown (24 arrangements) · focus star · mute. The × at the very end removes the session (frees its colour slot).
★ Focus (priority)
This is the answer to "Terminal C has the important message but Terminal A is 3 clips deep and B is another 3". Star Terminal C and its unplayed clips jump the queue — they play before any pending clip from other sessions. It never interrupts the clip that's currently speaking. Only one session can be starred at a time; clicking another star swaps focus. Active row picks up a subtle gold tint.
🔊 / 🔇 Mute
Silences one session completely: no synthesis, no file, no dot, no audio. Muted rows fade to 55 % and the terminal's statusline shows a 🔇 prefix next to its emoji. Use it for backgrounding a chatty agent while you focus elsewhere.
Playback precedence
When the player picks the next clip: (1) highlight-to-speak (hey jarvis / Ctrl+Shift+S) always wins · (2) unplayed clips from the ★ session · (3) oldest unplayed clip from any unmuted session.
Expanded row (chevron ▾)
Reveals per-session Voice (45 Edge Neural voices — pick one per terminal so you can tell them apart by ear) and 6 tri-state Speech-includes toggles: code blocks · inline code · URLs · headings · bullet markers · image alt-text. Each toggle is ● on / ● follow global / ○ off.
Global shortcuts
Ctrl+Shift+A show/hide toolbar · Ctrl+Shift+S read selection · Ctrl+Shift+J toggle mic · Ctrl+Shift+P pause/resume · Ctrl+Shift+O pause-only. All global, so they work from any app.

Trigger
Drag the bar within ~20 px of the top edge of the current screen. On release it snaps flush; the top corners go square so it reads as docked.
Same layout
Top-snap keeps the default two-row letterbox — all controls still fit. The snap is purely positional.
Persistence
The snap edge is written to ~/.terminal-talk/config.json (window.dock: "top") so the bar comes back to the same place on relaunch, even across display changes.
Multi-monitor
Snap uses the work-area of whichever display the bar is currently on — drag to another monitor and the next snap targets that monitor's edges.
Unsnapping
Drag away from the edge. The round pill shape returns the moment it's more than a few pixels clear of the top.
