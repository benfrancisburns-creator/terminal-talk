Terminal Talk — architecture

two flows · how audio gets from a wake word (or a Claude reply) to a coloured dot
Flow 1 · "hey jarvis" → highlighted text → dot

Wake-word Python daemon listens in the background. On match, Electron main reads the OS selection, strips markup, sends text to Edge TTS, and writes the resulting clip into the queue directory — the renderer tails it and appends a dot.

Flow 2 · Claude Code reply → auto-spoken

When Claude Code finishes a turn, its Stop hook POSTs the reply text. The registry stamps the session's colour + voice, then the same TTS pipeline lands a clip in the queue — producing a matching coloured dot on the bar.
