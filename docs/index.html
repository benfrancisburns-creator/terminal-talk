What you get

Six things that make Claude Code a hands-free tool

Every feature is opt-in at install. No telemetry anywhere.

🔊

Streaming auto-speak

Claude's responses are spoken as they're written. Audio starts ~2-3 s after Claude begins, not 6-24 s after the turn ends — sentences synthesise in parallel and a PreToolUse hook fires mid-response. Questions spoken first.

👂

"Hey jarvis" read-anything

Highlight text in any app — browser, PDF, VS Code, Slack — say the wake word or press Ctrl+Shift+S, hear it read. Runs offline on CPU.

🎨

Per-terminal identity + mute

Each Claude Code terminal gets a unique colour dot, statusline emoji, and optional voice. Mute any terminal with one click — its synthesis skips entirely, no audio, no queued clips.

📌

Dockable toolbar

Drag to any screen edge and it snaps flush. Left/right edges switch to a vertical column. Idle 15 s → shrinks to a slim strip and becomes click-through; hover to expand. Per-session colour assignments persist until you remove them.

🧹

Self-managing queue

Played clips auto-prune 20 s after they finish (configurable 3-600 s). Walking away? Toggle off and clips stack up for review on return. No manual housekeeping.

🔒

Free + private by default

Uses Microsoft Edge's neural voices (no API key needed). Offline wake word. Optional OpenAI TTS fallback if you want it. Zero accounts required.

Install

Three commands, under two minutes

  1. Install Python 3.10+ and Node.js 18+ if you don't already have them.
  2. Clone the repo and run install.ps1 — installs deps, pre-downloads the wake-word model, asks which hooks you want registered.
  3. Restart Claude Code. Every response will speak aloud. Say "hey jarvis" over highlighted text to read it.
PS> git clone https://github.com/benfrancisburns-creator/terminal-talk
PS> cd terminal-talk
PS> .\install.ps1
Windows 10/11 · Python 3.10+ · Node 18+ · Microphone
Mac and Linux ports tracked for v0.2. Uninstall with .\uninstall.ps1 — reverses everything.
Privacy

What leaves your machine — exactly

Nothing is sent anywhere without your explicit action. No telemetry. No analytics. No remote logging.

What you control

  • TTS text sent to Edge's neural voice endpoint (same one Edge browser uses)
  • Wake-word model download (one-off, from HuggingFace)
  • Optional: text sent to OpenAI TTS if you configure that fallback

What never happens

  • No audio is uploaded; wake-word runs offline on your CPU
  • No usage telemetry or analytics of any kind
  • No background network calls when idle
  • No account, no sign-up, no email required
FAQ

Likely questions

Is it really free, with no sign-up?

Yes. It uses Microsoft Edge's built-in neural voice endpoint (the same one Edge browser uses for "Read Aloud" — no API key required) and openWakeWord for offline wake-word detection. An OpenAI TTS fallback is optional if you'd rather pay for premium voices.

Why Windows-only at launch?

The toolbar and hooks rely on PowerShell + Windows-specific keybindings. Mac and Linux ports are the priority for v0.2. If you can help with a port, PRs are welcome.

Can I change the wake word from "hey jarvis"?

Yes — the wake model is a drop-in swap from the openWakeWord project. Documented in the README. You can also disable the wake word entirely and just use Ctrl+Shift+S.

Does it work with tools other than Claude Code?

The "hey jarvis / highlight and speak" part works with any app (browsers, PDFs, editors, Slack). The auto-speak piece is specific to Claude Code's Stop hook.

How do I uninstall?

Run .\uninstall.ps1. It reverses every change, restores your Claude Code settings from the backup created at install time, and removes the Startup shortcut.

What's with the coloured dots?

Each Claude Code terminal gets a unique dot colour on the toolbar and a matching emoji in its statusline, so when you've got five terminals open you can tell which one just spoke. You can also give each terminal its own voice and identify them by ear.
